# Go CLI 再設計 (概要)

> **ステータス**: 設計中  
> **作成日**: 2026-01-08  
> **対象**: `tools/` ディレクトリの Go への全面書き換え

## 概要

Python で実装されている CLI ツール群 (`tools/`) を Go (Kong CLI framework) で再実装する。

## 目的

| 現状の課題 | 解決策 |
|-----------|-------|
| バイナリ配布不可 (Python + pip + venv 必須) | シングルバイナリ |
| containerd/Docker との連携が間接的 | ネイティブ SDK 利用 |
| 依存管理が複雑 | go.mod で完結 |
| クロスプラットフォーム対応困難 | `GOOS`/`GOARCH` でビルド |

## 設計原則

> **既存の SAM プロジェクトを壊さない**
>
> このツールは既存の SAM テンプレートを入力として受け取り、
> ソースプロジェクトを一切変更せずに動作する。
>
> ただし運用上の実装として、`generator.yml` と `.esb/` は
> テンプレート同階層に生成する。これらは生成物として扱い、
> `.gitignore` に追加して管理対象外とする前提。
